Why Visualise?

  • debug complex systems during development:
    • physics middleware: suspension spring compression over race course length
    • game AI: why are characters walking into a wall at point X?
    • what else?
  • explain a phenomenon from a simulated experiment:
    • CS AI: at what rate are agents learning compared to previous algorithm/environment?
    • how will an epidemic spread through a population?
  • provide extra interactive information
    • medical imaging from scanner-obtained data
    • augmented reality (satellite image, HUD)
  • we need to choose our visualisation pipeline carefully
  • data sets to use, visualisation methods, standards, tools

Visualisation Options

  • static graphs or figures
    • suitable for print media (papers, journals, theses)
    • isolate 1 type of data - easy to appreciate
    • illustrate relationships between data sets
  • interactive visualisation
    • virtual camera
    • explore complex virtual world to observe phenomena from different view-points
    • interrupt to dump interesting data sets or to record stills or video
  • video and screen captures
    • not valid as scientific results - noise, no error
    • replay interesting phenomena in appreciable way
    • accompany other visualisations
  • sometimes it's easier to read data in a table

How to Select Visualisations

  • Ask yourself:

    Does this visualisation communicate information:

    A. more clearly?
    B. less clearly?

How to Select Visualisations

  • start with a table if possible
  • then choose the most appropriate graphs to display data using the more clear/less clear question.
  • use all scientific standards available to increase quality:
    • error measurements or measurements of uncertainty
    • scales, titles, lables, number of points
    • control plots, comparisons
  • then consider interactive displays and video to support the measured results

error measurements and uncertainty

  • scientific measurements are repeated n times
  • we must represent uncertainty visually
  • use standard error or standard deviation for each point or bar
  • remember: sample standard deviation
  • and standard error is std dev divided by square root of n
  • .: bigger n → small standard error
  • if an error bar is large: collect more data

interpreting error bars

  • SE bars → 68% confidence that the true mean lies inside the error bars
  • 2X SD bars are called the 95% confidence interval
  • with SE bars two points are significantly different if there is at least 1/2 bar gap between the error bars.
  • 95% confidence interval bars are longer. Less than 1/4 bar of overlap corresponds to P < 0.05
  • always explain the statistics used in the text or caption
    • n
    • if standard deviation or standard error is used
    • discuss any unsual points or trends

heat maps

heat maps

  • illustrate frequency of something happening in geographic regions
  • break environment into nodes or grid cells
  • when event occurs, add value with large weight to closest node, and smaller weight to nearby nodes
  • for real-time, reduce (decay) values over time
  • display using colour range or 3D map peaks - what are the drawbacks of each?

a good data representation has

  • axes with consistent scales
  • axis labels with units given
  • clear plots with control plot or comparison
  • no clutter, minimal noise
  • supporting caption. explain what the reader is supposed to see. explain any interesting areas.
  • error measurements where possible. explain statistics; standard error or deviation or 95% confidence, size of n
  • key to explain heat map colour ranges
  • key to explain what the different data sets are
  • always provide and explain any formulae used - how is the distance between nodes or the colour calculated?
  1. can the viewer determine the difference, in units, between data points in your visualisation?
  2. have you provided enough information so that another scientist can visualise their own data set and compare results with you?
